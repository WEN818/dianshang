## 第一部分：申请服务器

### 1.1 选择云服务商

在中国，推荐使用以下云服务商（按推荐顺序）：

1. **阿里云**（https://www.aliyun.com）- 国内最大，文档完善
2. **腾讯云**（https://cloud.tencent.com）- 性价比高，学生优惠多
3. **华为云**（https://www.huaweicloud.com）- 企业级服务
4. **百度云**（https://cloud.baidu.com）- 适合AI相关项目

**新手推荐：腾讯云**（学生认证后价格便宜，界面友好）

### 1.2 以腾讯云为例，申请服务器详细步骤

#### 步骤 1：注册账号

1. 访问 https://cloud.tencent.com
2. 点击右上角"注册"
3. 使用手机号或邮箱注册
4. 完成实名认证（需要身份证）

#### 步骤 2：学生认证（可选，但强烈推荐）

1. 登录后，点击右上角头像 → "学生认证"
2. 填写学生信息（学信网验证）
3. 认证成功后，可享受学生优惠价格

#### 步骤 3：购买云服务器

1. 登录控制台，点击"云服务器 CVM" → "新建"
2. 选择配置（**新手推荐配置**）：
   - **地域**：选择离您最近的（如：北京、上海、广州）
   - **机型**：选择"标准型 S5"或"标准型 SA2"
   - **镜像**：选择 **Ubuntu 20.04 LTS** 或 **Ubuntu 22.04 LTS**（推荐）
   - **系统盘**：40GB SSD（默认即可）
   - **公网带宽**：1Mbps（够用，后续可升级）
   - **购买时长**：1个月（学生优惠通常很便宜）

3. **重要设置**：
   - **登录方式**：选择"立即设置密码"
   - **用户名**：默认是 `ubuntu`（Ubuntu系统）或 `root`（CentOS系统）
   - **密码**：设置一个强密码（**请务必记住！**）
     - 建议包含：大小写字母、数字、特殊字符
     - 例如：`MyServer2024!@#`

4. 点击"立即购买" → 确认订单 → 支付

#### 步骤 4：获取服务器信息

购买成功后，在控制台可以看到：

- **公网 IP**：例如 `123.456.789.123`（这是访问服务器的地址）
- **内网 IP**：例如 `10.0.0.5`（服务器内部使用）
- **用户名**：`ubuntu`（Ubuntu系统）
- **密码**：您刚才设置的密码

**请记录这些信息，后续会用到！**

#### 步骤 5：配置安全组（防火墙规则）

1. 在服务器列表中，点击您的服务器名称
2. 点击"安全组"标签
3. 点击"修改规则"
4. 添加以下规则（点击"添加规则"）：

| 类型 | 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|------|
| 入站规则 | TCP | 22 | 0.0.0.0/0 | SSH连接（必须） |
| 入站规则 | TCP | 80 | 0.0.0.0/0 | HTTP（可选） |
| 入站规则 | TCP | 443 | 0.0.0.0/0 | HTTPS（可选） |
| 入站规则 | TCP | 8080 | 0.0.0.0/0 | 应用端口（必须） |

5. 点击"完成"

**注意**：如果您的服务器提供商是阿里云，需要在"安全组"中配置；如果是华为云，在"网络ACL"中配置。

## 第二部分：连接服务器

### 2.1 Windows 系统连接服务器

#### 方法一：使用 PowerShell（推荐，Windows 10/11自带）

1. 按 `Win + X`，选择"Windows PowerShell"或"终端"
2. 输入以下命令（替换为您的IP和用户名）：

```bash
ssh ubuntu@123.456.789.123
```

3. 第一次连接会提示确认，输入 `yes` 并按回车
4. 输入密码（输入时不会显示，直接输入后按回车）

**如果连接成功，您会看到类似这样的提示：**
```
Welcome to Ubuntu 22.04 LTS...
ubuntu@VM-0-1-ubuntu:~$
```

#### 方法二：使用 PuTTY（图形界面，更友好）

1. 下载 PuTTY：https://www.putty.org/
2. 安装并打开 PuTTY
3. 填写信息：
   - **Host Name**：`123.456.789.123`（您的公网IP）
   - **Port**：`22`
   - **Connection type**：选择 `SSH`
4. 点击"Open"
5. 输入用户名：`ubuntu`
6. 输入密码（输入时不会显示）

### 2.2 Mac/Linux 系统连接服务器

打开终端（Terminal），输入：

```bash
ssh ubuntu@123.456.789.123
```

输入密码即可连接。

### 2.3 连接成功后

您应该能看到类似这样的命令行提示符：

```bash
ubuntu@VM-0-1-ubuntu:~$
```

这表示您已经成功连接到服务器了！

**提示**：如果连接失败，请检查：
- IP地址是否正确
- 安全组是否开放了22端口
- 密码是否正确
- 服务器是否已启动（在控制台查看状态）

## 第三部分：安装必要软件

### 3.1 更新系统（重要！）

连接服务器后，首先更新系统：

```bash
# 更新软件包列表
sudo apt update

# 升级已安装的软件包
sudo apt upgrade -y
```

**说明**：
- `sudo`：以管理员权限执行命令
- `apt`：Ubuntu的包管理器
- `update`：更新软件包列表
- `upgrade`：升级软件包
- `-y`：自动确认，不需要手动输入yes

### 3.2 安装 Docker

Docker 是一个容器化平台，可以让应用运行在隔离的环境中。

#### 步骤 1：安装 Docker

复制以下命令，一行一行执行：

```bash
# 安装必要的依赖
sudo apt install -y ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG 密钥
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 添加 Docker 仓库（使用阿里云镜像，速度更快）
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://mirrors.aliyun.com/docker-ce/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 更新软件包列表
sudo apt update

# 安装 Docker
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

#### 步骤 2：启动 Docker 服务

```bash
# 启动 Docker
sudo systemctl start docker

# 设置 Docker 开机自启
sudo systemctl enable docker
```

#### 步骤 3：将当前用户添加到 docker 组（避免每次都用sudo）

```bash
# 将当前用户添加到 docker 组
sudo usermod -aG docker $USER

# 重新加载用户组（或重新连接服务器）
newgrp docker
```

#### 步骤 4：验证 Docker 安装

```bash
# 检查 Docker 版本
docker --version

# 运行测试容器
docker run hello-world
```

**如果看到 "Hello from Docker!" 的提示，说明 Docker 安装成功！**

### 3.3 安装 Docker Compose

Docker Compose 用于管理多个容器的应用。

```bash
# 下载 Docker Compose（使用国内镜像）
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 如果上面的命令下载慢，可以使用这个（使用代理镜像）
# sudo curl -L "https://ghproxy.com/https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 添加执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

**如果显示版本号（如 `Docker Compose version v2.x.x`），说明安装成功！**

### 3.4 安装 Git（用于下载项目代码）

```bash
sudo apt install -y git
```

## 第四部分：上传项目文件

### 4.1 方法一：使用 Git（推荐，如果项目在GitHub/Gitee）

#### 步骤 1：创建项目目录

```bash
# 创建目录
sudo mkdir -p /opt/network-homework

# 设置权限
sudo chown -R $USER:$USER /opt/network-homework

# 进入目录
cd /opt/network-homework
```

#### 步骤 2：克隆项目

**如果项目在 GitHub：**

```bash
# 克隆项目（替换为您的仓库地址）
git clone https://github.com/your-username/your-repo-name.git .

# 或者如果项目在 Gitee（国内访问更快）
git clone https://gitee.com/your-username/your-repo-name.git .
```

**如果项目在本地，使用方法二。**

### 4.2 方法二：使用 SCP 上传（从本地电脑上传）

#### Windows 系统：

1. 打开 PowerShell（在项目目录下）
2. 执行以下命令（替换为您的信息）：

```powershell
# 上传整个项目文件夹
scp -r network-homework ubuntu@123.456.789.123:/opt/
```

3. 输入密码

#### Mac/Linux 系统：

```bash
# 在本地项目目录下执行
scp -r network-homework ubuntu@123.456.789.123:/opt/
```

#### 然后在服务器上：

```bash
# 进入项目目录
cd /opt/network-homework
```

### 4.3 方法三：使用 WinSCP（Windows图形界面工具，推荐新手）

1. 下载 WinSCP：https://winscp.net/
2. 安装并打开 WinSCP
3. 填写连接信息：
   - **文件协议**：SFTP
   - **主机名**：`123.456.789.123`（您的IP）
   - **端口号**：`22`
   - **用户名**：`ubuntu`
   - **密码**：您的服务器密码
4. 点击"登录"
5. 左侧是本地文件，右侧是服务器文件
6. 将 `network-homework` 文件夹拖到右侧的 `/opt/` 目录

### 4.4 验证文件已上传

```bash
# 查看项目文件
cd /opt/network-homework
ls -la

# 应该能看到以下文件：
# - compose.yaml
# - Dockerfile
# - deploy.sh
# - pom.xml
# - src/

```

## 第五部分：配置和部署

### 5.1 修改数据库密码（重要！）

**默认密码不安全，必须修改！**

```bash
# 进入项目目录
cd /opt/network-homework

# 编辑配置文件
nano compose.yaml
```

**使用 nano 编辑器：**
- 方向键移动光标
- 找到以下两行（大约在第9-10行）：
  ```yaml
  - MYSQL_PASSWORD=network_password
  - MYSQL_ROOT_PASSWORD=root_password
  ```
- 修改为强密码，例如：
  ```yaml
  - MYSQL_PASSWORD=MyStrongPass123!@#
  - MYSQL_ROOT_PASSWORD=MyRootPass456!@#
  ```
- 同时修改第15行的健康检查密码（与MYSQL_ROOT_PASSWORD一致）：
  ```yaml
  -proot_password
  ```
  改为：
  ```yaml
  -pMyRootPass456!@#
  ```
- 按 `Ctrl + O` 保存，按 `Enter` 确认
- 按 `Ctrl + X` 退出

**提示**：如果不会用 nano，也可以使用 `vi` 编辑器，或者下载到本地修改后重新上传。

### 5.2 配置防火墙（Ubuntu系统）

```bash
# 检查防火墙状态
sudo ufw status

# 如果防火墙未启用，执行以下命令：
sudo ufw allow 22/tcp    # SSH端口（必须）
sudo ufw allow 80/tcp    # HTTP端口（可选）
sudo ufw allow 443/tcp   # HTTPS端口（可选）
sudo ufw allow 8080/tcp  # 应用端口（必须）

# 启用防火墙
sudo ufw enable

# 确认规则已添加
sudo ufw status
```

**注意**：如果您的云服务商有安全组设置（如腾讯云、阿里云），安全组已经配置好了，这一步可以跳过。但双重保护更安全。

### 5.3 部署应用

#### 方法一：使用部署脚本（推荐）

```bash
# 确保在项目目录
cd /opt/network-homework

# 给脚本添加执行权限
chmod +x deploy.sh

# 运行部署脚本
./deploy.sh
```

脚本会自动：
1. 检查 Docker 和 Docker Compose 是否安装
2. 停止旧容器（如果有）
3. 构建并启动新容器
4. 显示部署状态

#### 方法二：手动部署

```bash
# 确保在项目目录
cd /opt/network-homework

# 停止旧容器（如果有）
docker-compose down

# 构建并启动容器
docker-compose up -d --build
```

**参数说明**：
- `up`：启动服务
- `-d`：后台运行（detached mode）
- `--build`：重新构建镜像

### 5.4 等待服务启动

部署需要几分钟时间，因为需要：
1. 下载 MySQL 镜像
2. 构建应用镜像
3. 启动数据库
4. 启动应用

**请耐心等待 2-5 分钟。**

## 第六部分：验证和访问

### 6.1 检查容器状态

```bash
# 查看容器运行状态
docker-compose ps
```

**应该看到两个容器都在运行（Status 显示 Up）：**
- `mysql-network-homework` - MySQL数据库
- `network-homework-app` - 应用服务

如果状态不是 `Up`，继续查看日志。

### 6.2 查看日志

```bash
# 查看所有服务的日志
docker-compose logs

# 查看应用日志（实时）
docker-compose logs -f app

# 查看数据库日志
docker-compose logs -f mysql
```

**正常情况应该看到：**
- MySQL：`ready for connections`
- App：`Started NetworkHomeworkApplication` 或类似信息

**如果有错误，请查看 [常见问题解答](#常见问题解答) 部分。**

### 6.3 测试访问

#### 在服务器上测试：

```bash
# 测试本地访问
curl http://localhost:8080
```

如果返回 HTML 内容，说明应用正常运行。

#### 在本地浏览器访问：

1. 打开浏览器
2. 访问：`http://您的服务器IP:8080`
   - 例如：`http://123.456.789.123:8080`

**如果能看到登录页面，恭喜您部署成功！** 🎉

### 6.4 配置自动备份和监控（可选，但推荐）

```bash
# 进入项目目录
cd /opt/network-homework

# 给脚本添加执行权限

# 配置定时监控（每5分钟检查一次）
(crontab -l 2>/dev/null; echo "*/5 * * * * /opt/network-homework/monitor.sh") | crontab -

crontab -l
```

## 常见问题解答

### Q1: 连接服务器时提示 "Connection refused"

**原因**：服务器未启动或安全组未开放22端口

**解决方法**：
1. 在云服务商控制台检查服务器状态，确保是"运行中"
2. 检查安全组是否开放了22端口
3. 确认IP地址是否正确

### Q2: Docker 镜像拉取失败（failed to resolve reference）

**错误信息**：
- `failed to resolve reference "docker.io/library/mysql:8.0"` 
- `no such host`
- `403 Forbidden`（镜像源不可用）

**原因**：Docker 镜像加速器配置问题、DNS 解析失败或镜像源不可用

**解决方法**：

#### 方法一：使用可靠的镜像源（推荐，解决 403 错误）

```bash
# 配置更可靠的镜像源
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json > /dev/null <<'EOF'
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://mirror.baidubce.com",
    "https://docker.nju.edu.cn",
    "https://dockerproxy.com"
  ],
  "dns": ["8.8.8.8", "114.114.114.114", "223.5.5.5"]
}
EOF

# 重启 Docker
sudo systemctl daemon-reload
sudo systemctl restart docker

# 等待几秒
sleep 3

# 验证配置
docker info | grep -A 10 "Registry Mirrors"
```

#### 方法二：使用云服务商专属镜像加速器（最稳定）

**如果您使用的是腾讯云服务器：**

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json > /dev/null <<'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com"
  ],
  "dns": ["8.8.8.8", "114.114.114.114"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

**如果您使用的是阿里云服务器：**

```bash
# 登录阿里云控制台，在"容器镜像服务" -> "镜像加速器"获取专属地址
# 或者使用公共地址：
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json > /dev/null <<'EOF'
{
  "registry-mirrors": [
    "https://registry.cn-hangzhou.aliyuncs.com"
  ],
  "dns": ["8.8.8.8", "114.114.114.114"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

#### 方法三：手动拉取镜像后再部署

如果镜像加速器都不行，可以尝试手动拉取：

```bash
# 尝试从不同源拉取 MySQL 镜像
docker pull mysql:8.0

# 如果上面失败，尝试使用完整镜像地址
docker pull docker.mirrors.ustc.edu.cn/library/mysql:8.0

# 或者直接使用官方源（如果网络允许）
docker pull docker.io/library/mysql:8.0

# 拉取成功后，再执行部署
cd /opt/network-homework
docker-compose up -d --build
```

#### 方法四：使用代理或直接连接（如果其他方法都失败）

```bash
# 临时使用官方源（不推荐，可能很慢）
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json > /dev/null <<'EOF'
{
  "dns": ["8.8.8.8", "114.114.114.114", "223.5.5.5"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker

# 然后尝试拉取（可能需要较长时间）
docker pull mysql:8.0
```

#### 方法五：如果 DNS 有问题，先修复 DNS

```bash
# 编辑 DNS 配置
sudo nano /etc/systemd/resolved.conf

# 取消注释并修改以下行：
# DNS=8.8.8.8 114.114.114.114 223.5.5.5
```

### Q2-1: 脚本执行错误（bad interpreter: No such file or directory）

**错误信息**：`./deploy.sh: /bin/bash^M: bad interpreter: No such file or directory`

**原因**：文件使用了 Windows 行尾符（CRLF），Linux 需要 Unix 行尾符（LF）

**解决方法**：

**方法一：使用 sed 命令修复（推荐，无需安装额外软件）**

```bash
# 进入项目目录
cd /opt/network-homework

# 修复所有 .sh 脚本文件的行尾符
sed -i 's/\r$//' deploy.sh
sed -i 's/\r$//' backup.sh
sed -i 's/\r$//' monitor.sh
sed -i 's/\r$//' fix-docker-mirror.sh

# 或者一次性修复所有脚本
find . -name "*.sh" -type f -exec sed -i 's/\r$//' {} \;

# 添加执行权限
chmod +x *.sh

# 验证修复
file deploy.sh
# 应该显示：deploy.sh: Bourne-Again shell script, ASCII text executable
```

**方法二：安装 dos2unix 工具（更专业）**

```bash
# 安装 dos2unix
sudo apt update
sudo apt install -y dos2unix

# 转换文件
cd /opt/network-homework
dos2unix deploy.sh backup.sh monitor.sh

# 添加执行权限
chmod +x *.sh
```

**方法三：使用 tr 命令**

```bash
cd /opt/network-homework
tr -d '\r' < deploy.sh > deploy.sh.tmp && mv deploy.sh.tmp deploy.sh
chmod +x deploy.sh
```

**验证修复**：

```bash
# 检查文件类型
file deploy.sh

# 应该显示类似：deploy.sh: Bourne-Again shell script, ASCII text executable
# 而不是：deploy.sh: Bourne-Again shell script, ASCII text executable, with CRLF line terminators

# 尝试执行
./deploy.sh
```

**预防措施**：以后上传文件到 Linux 服务器时，建议：
1. 使用 Git 克隆（Git 会自动处理行尾符）
2. 使用 WinSCP 上传时，在"传输设置"中选择"文本模式"
3. 在本地使用 Git 配置：`git config --global core.autocrlf input`

### Q3: 容器启动失败，查看日志显示端口被占用

**原因**：8080端口已被其他程序占用

**解决方法**：

```bash
# 查看8080端口占用
sudo netstat -tulpn | grep 8080

# 或者
sudo lsof -i :8080

# 停止占用端口的程序，或修改 compose.yaml 中的端口映射
# 例如改为 8081:8080
```

### Q4: 应用无法访问，但容器运行正常

**原因**：可能是防火墙或安全组未开放8080端口

**解决方法**：
1. 检查服务器防火墙：`sudo ufw status`
2. 检查云服务商安全组是否开放8080端口
3. 在服务器上测试：`curl http://localhost:8080`

### Q5: 数据库连接失败

**原因**：数据库密码不匹配或数据库未启动

**解决方法**：
1. 检查数据库容器是否运行：`docker-compose ps`
2. 查看数据库日志：`docker-compose logs mysql`
3. 确认 `compose.yaml` 中的密码配置一致
4. 等待数据库完全启动（通常需要30-60秒）

### Q6: 忘记服务器密码

**解决方法**：
1. 在云服务商控制台重置密码
2. 重启服务器使新密码生效
